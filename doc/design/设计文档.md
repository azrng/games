# 游戏平台设计文档

## 五子棋移动端独立页面优化（2026-05-13）

### 架构设计
- 页面结构：保留 `gomoku-advanced/index.html` 作为 PC 页面，新增 `gomoku-advanced/mobile.html` 作为移动端专用页面。
- 数据流向：移动端和 PC 端共用 `gomoku-advanced/script.js` 与 `gomoku-advanced/ai-worker.js`，避免复制五子棋规则、AI 通信和棋盘渲染逻辑。
- 入口策略：首页游戏清单为五子棋补充 `mobilePath`，外层 `js/app.js` 根据视口宽度选择 PC 或移动端页面加载到 iframe。
- 组件与交互：移动端页面使用顶部状态栏、棋盘主舞台、底部固定操作栏和设置抽屉；设置控件沿用原有 id，保证共享脚本可直接绑定。

### 功能需求
- 页面列表：
  - PC 页面：`gomoku-advanced/index.html`，保持现有双栏布局。
  - 移动端页面：`gomoku-advanced/mobile.html`，面向手机首屏体验。
- 功能点：
  - 开始、悔棋、认输可在移动端底部操作栏直接使用，移动端不展示提示入口。
  - 当前状态、步数、当前回合展示在顶部状态栏。
  - 电脑先手、难度、主题、禁手规则移动到设置抽屉。
  - 横竖屏或窗口尺寸变化时，棋盘自动重新计算尺寸。
- 业务规则：
  - 不改变五子棋规则、AI 算法、禁手判定与 PC 端提示逻辑。
  - 移动端触摸落子必须与视觉交叉点一致。
- 权限矩阵：本项目为静态游戏页，无登录权限区分，所有用户可访问全部五子棋功能。

### 界面原型

```text
移动端五子棋
+--------------------------------+
| 五子棋            设置  状态    |
| 当前回合 / 步数 / AI 状态       |
+--------------------------------+
|                                |
|        15 x 15 棋盘             |
|     轻触交叉点完成落子          |
|                                |
+--------------------------------+
| 开始 | 悔棋 | 认输              |
+--------------------------------+

设置抽屉
+--------------------------------+
| 对局设置                    关闭 |
| [ ] 电脑先手                    |
| 难度    [简单 v]                |
| 主题    [经典 v]                |
| [ ] 禁手规则                    |
+--------------------------------+
```

- loading：棋盘区域显示 `AI 思考中...` 遮罩。
- empty：未开始时顶部状态展示“点击‘开始’进入游戏”。
- error：Worker 或脚本异常时由浏览器控制台暴露，当前静态页不新增错误页。

### 交互说明
- 用户在首页进入五子棋时，若视口宽度小于等于 640px，加载 `mobile.html`；否则加载 `index.html`。
- 点击“设置”打开底部抽屉，再次点击遮罩、关闭按钮或抽屉内“完成”关闭。
- 点击电脑先手和禁手规则的开关滑块即可切换；点击“开始”时脚本会读取当前电脑先手和难度值，禁手规则变更后立即影响棋盘禁手标记与落子校验。
- 点击棋盘时，脚本以 canvas 实际显示矩形计算落子坐标，避免 CSS 缩放造成偏移。
- 点击“开始”后进入游戏，AI 思考期间禁用相关按钮并显示加载遮罩。
- 窗口 resize 时首页会按当前视口切换五子棋 PC / 移动端页面；游戏内棋盘同步重新计算尺寸。

## 迷宫冒险布局与主题优化（2026-05-13）

### 架构设计
- 页面结构：保留 `maze/index.html` 单文件游戏实现，继续使用原有递归回溯迷宫生成、键盘移动、通关结算和桌面端限制逻辑。
- 布局策略：新增 `game-sidebar` 作为桌面端固定侧栏，整合玩法说明与关卡、步数、耗时、尺寸、模式统计；`game-container` 独立作为迷宫舞台。
- 主题策略：迷宫页面统一到游戏平台浅色体系，使用蓝色作为主操作色、绿色作为探索/已访问状态、琥珀色作为终点提示，避免原深蓝霓虹主题与其他游戏割裂。
- 入口文案：统一清单和说明文档改为桌面端专用，不再宣称移动端方向键和缩放。

### 功能需求
- 保持三档难度、关卡递增、路径访问痕迹、通关进入下一关等玩法不变。
- 迷宫主体需要避让左侧侧栏，避免 HUD 和统计浮层遮挡可视区域。
- 难度选择和通关弹层使用浅色面板，与首页和五子棋的视觉语言保持一致。
- 移动设备访问时继续展示桌面端专用提示。

### 界面原型

```text
----------------------+--------------------------------+
| 迷宫冒险             |                                |
| 穿过迷宫，找到伙伴   |          迷宫舞台               |
| 目标说明             |     玩家 / 路径 / 终点          |
|                      |                                |
| 关卡  步数           |                                |
| 耗时  尺寸           |                                |
| 模式                 |                                |
+----------------------+--------------------------------+
```

### 交互说明
- 玩家在难度弹层选择简单、中等、困难后开始游戏。
- 游戏中使用方向键或 `WASD` 移动，统计区实时更新关卡、步数、耗时、尺寸与模式。
- 侧栏占用的屏幕区域会参与安全区域计算，迷宫自动居中到剩余舞台空间。
- 通关弹层展示本关耗时和步数，点击按钮进入下一关。

## 2048 游戏进度存储（2026-05-13）

### 架构设计
- 复用现有 `AppStore`（`window.AppStore`）存储工具，通过 `setGameJSON`/`getGameJSON` 操作 localStorage，key 为 `game_2048_game_state`。
- 仅修改 `2048/script.js`，不新增文件。

### 功能需求
- 每次移动成功后，将棋盘状态（grid、score、over、won、keepPlayingAfterWin）保存到 localStorage。
- 页面加载时，优先从 localStorage 恢复存档；无有效存档或游戏已结束时开始新游戏。
- 游戏结束（Game Over 或胜利未继续）不保存进度，下次打开重新开始。
- 点击"新游戏"时清除已保存的状态。

### 交互说明
- 用户无需手动操作，进度自动保存和恢复。
- 关闭页面再打开，棋盘、分数恢复到上一步操作后的状态。

## 色觉挑战（Kuku Kube）（2026-05-13）

### 架构设计
- 页面结构：新增 `color-challenge/index.html` 作为纯移动端单页面游戏入口，配套 `style.css` 和 `script.js`。
- 数据流向：页面启动后自动生成第一轮，游戏状态保存在内存中；最高分和最长存活轮数写入 `localStorage`，键名前缀使用需求文档指定的 `color-challenge-`。
- 渲染方式：使用 CSS Grid 和 `background-color` 渲染色块，不使用 Canvas；脚本暴露 `window.ColorChallenge` 中的少量纯函数，便于 smoke test 校验难度与存储规则。
- 首页接入：在 `data/games.js` 中新增色觉挑战条目，移动端和桌面端均指向同一入口；页面自身在桌面宽度展示移动端提示。

### 功能需求
- 进入页面自动开局，无需点击开始。
- 每轮生成一个 N×N 色块网格，只有一个色块颜色不同。
- 轮次 1-5 使用 2×2，6-10 使用 3×3，11-20 使用 4×4，21-30 使用 5×5，31 轮起使用 6×6。
- 正确点击后得分 +1 并进入下一轮；错误点击后结束游戏并展示本局得分、存活轮数、历史最高分和最长轮数。
- 色差随轮次递增逐渐缩小，但保留最小可辨识下限。
- 快速连续点击时每轮只响应第一次判定。

### 界面原型

```text
+----------------------------------+
| 色觉挑战                  12秒    |
| 轮次 得分 最高分 最长轮数         |
+----------------------------------+
|                                  |
|          色块 CSS Grid            |
|     找出唯一不同色并轻触          |
|                                  |
+----------------------------------+
| 纯色觉游戏可能不适合所有用户       |
+----------------------------------+
```

### 交互说明
- 玩家点击正确色块后，色块短暂缩放高亮，再平滑进入下一轮。
- 玩家点击错误色块后，网格抖动并弹出结算面板。
- 点击“再来一局”清空当前状态并从第 1 轮重新开始。
- 横竖屏切换时网格通过 CSS `aspect-ratio` 保持正方形并适配可用宽度。

## 数字华容道（Sliding Puzzle）（2026-05-13）

### 架构设计
- 页面结构：新增 `sliding-puzzle/index.html` 作为移动端单页面游戏入口，配套 `style.css` 和 `script.js`。
- 数据流向：棋盘状态、步数、计时和撤销栈保存在页面内存；每种规格的历史最少步数与最短用时写入 `localStorage`，键名前缀为 `sliding-puzzle-`。
- 生成策略：从目标完成状态出发，随机执行合法移动进行逆向打乱，避免生成无解局面；3×3、4×4、5×5 分别使用不同打乱步数。
- 渲染方式：使用 CSS Grid 渲染棋盘，数字方块为按钮，空位为占位格；脚本暴露 `window.SlidingPuzzle` 中的纯函数用于 smoke test 校验移动、可解性、打乱和记录。
- 首页接入：在 `data/games.js` 中新增数字华容道条目，移动端和桌面端均指向同一入口；页面自身在桌面宽度展示移动端提示。

### 功能需求
- 首次进入展示规格选择，支持 3×3 入门、4×4 经典、5×5 挑战。
- 选择规格后自动打乱并开始游戏，计时从第一步有效移动开始。
- 与空位相邻的方块可点击或触摸滑动进入空位；不相邻方块无响应。
- 每次有效移动步数 +1，并将移动前棋盘压入撤销栈。
- 点击“撤销”回退一步，撤销到初始打乱状态后按钮禁用。
- 点击“打乱”重新打乱当前规格；点击“重置”确认后返回规格选择。
- 棋盘归位后展示通关结算，包含规格、步数、用时、历史最少步数和最短用时。

### 界面原型

```text
+----------------------------------+
| 数字华容道              规格      |
| 规格  步数  时间                  |
| 最少步数 / 最短用时               |
+----------------------------------+
|                                  |
|       N x N 数字滑块棋盘          |
|                                  |
+----------------------------------+
|  撤销  |  打乱  |  重置           |
+----------------------------------+
```

### 交互说明
- 规格选择面板默认展示，选择后进入游戏。
- 点击相邻方块会自动滑入空位；触摸滑动超过阈值时同样尝试移动该方块。
- 重置当前棋盘前使用浏览器确认框，避免误触。
- 通关后可选择“再来一局”沿用当前规格，或“换个规格”回到规格选择。
