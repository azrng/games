# 游戏平台设计文档

## 五子棋移动端独立页面优化（2026-05-13）

### 架构设计
- 页面结构：保留 `gomoku-advanced/index.html` 作为 PC 页面，新增 `gomoku-advanced/mobile.html` 作为移动端专用页面。
- 数据流向：移动端和 PC 端共用 `gomoku-advanced/script.js` 与 `gomoku-advanced/ai-worker.js`，避免复制五子棋规则、AI 通信和棋盘渲染逻辑。
- 入口策略：首页游戏清单为五子棋补充 `mobilePath`，外层 `js/app.js` 根据视口宽度选择 PC 或移动端页面加载到 iframe。
- 组件与交互：移动端页面使用顶部状态栏、棋盘主舞台、底部固定操作栏和设置抽屉；设置控件沿用原有 id，保证共享脚本可直接绑定。

### 功能需求
- 页面列表：
  - PC 页面：`gomoku-advanced/index.html`，保持现有双栏布局。
  - 移动端页面：`gomoku-advanced/mobile.html`，面向手机首屏体验。
- 功能点：
  - 开始、悔棋、认输可在移动端底部操作栏直接使用，移动端不展示提示入口。
  - 当前状态、步数、当前回合展示在顶部状态栏。
  - 电脑先手、难度、主题、禁手规则移动到设置抽屉。
  - 横竖屏或窗口尺寸变化时，棋盘自动重新计算尺寸。
- 业务规则：
  - 不改变五子棋规则、AI 算法、禁手判定与 PC 端提示逻辑。
  - 移动端触摸落子必须与视觉交叉点一致。
- 权限矩阵：本项目为静态游戏页，无登录权限区分，所有用户可访问全部五子棋功能。

### 界面原型

```text
移动端五子棋
+--------------------------------+
| 五子棋            设置  状态    |
| 当前回合 / 步数 / AI 状态       |
+--------------------------------+
|                                |
|        15 x 15 棋盘             |
|     轻触交叉点完成落子          |
|                                |
+--------------------------------+
| 开始 | 悔棋 | 认输              |
+--------------------------------+

设置抽屉
+--------------------------------+
| 对局设置                    关闭 |
| [ ] 电脑先手                    |
| 难度    [简单 v]                |
| 主题    [经典 v]                |
| [ ] 禁手规则                    |
+--------------------------------+
```

- loading：棋盘区域显示 `AI 思考中...` 遮罩。
- empty：未开始时顶部状态展示“点击‘开始’进入游戏”。
- error：Worker 或脚本异常时由浏览器控制台暴露，当前静态页不新增错误页。

### 交互说明
- 用户在首页进入五子棋时，若视口宽度小于等于 640px，加载 `mobile.html`；否则加载 `index.html`。
- 点击“设置”打开底部抽屉，再次点击遮罩、关闭按钮或抽屉内“完成”关闭。
- 点击棋盘时，脚本以 canvas 实际显示矩形计算落子坐标，避免 CSS 缩放造成偏移。
- 点击“开始”后进入游戏，AI 思考期间禁用相关按钮并显示加载遮罩。
- 窗口 resize 时首页会按当前视口切换五子棋 PC / 移动端页面；游戏内棋盘同步重新计算尺寸。
