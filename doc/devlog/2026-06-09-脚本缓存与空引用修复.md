# 脚本缓存与空引用修复

## 本次目标

- 修复浏览器仍加载旧脚本导致的 `Cannot set properties of null (setting 'hidden')` 报错。
- 避免已删除猜拳弹窗后残留脚本缓存继续引用旧 DOM。

## 核心改动

- 为动物翻棋页面的 `style.css` 和 `script.js` 增加版本号 query，强制 Live Server 和浏览器拉取新资源。
- 对结果弹窗 `resultModal.hidden` 操作增加空值保护，降低 DOM 变更后的崩溃风险。

## 修改文件

- `animal-flip-chess/index.html`
- `animal-flip-chess/script.js`
- `TASK.md`

## 校验情况

- 已执行：`node animal-flip-chess/smoke.test.js`
- 已执行：`node --check animal-flip-chess/script.js`
- 已执行：`Invoke-WebRequest http://127.0.0.1:5509/animal-flip-chess/script.js?v=20260609-2`，返回 200。
- 已执行：读取本地服务返回的 HTML，确认包含 `script.js?v=20260609-2` 与 `style.css?v=20260609-2`。

## 风险或遗留项

- 浏览器控制台中的旧报错需要刷新页面后才会消失；若仍看到旧行号，需强制刷新或关闭后重开页面。
