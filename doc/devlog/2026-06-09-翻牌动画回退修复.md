# 翻牌动画回退修复

## 本次目标

- 修复机器人翻牌时，已经翻开的牌重新被背面盖住再翻开的视觉问题。

## 核心改动

- 为棋子增加稳定 `id`，用于识别已经展示过的翻牌状态。
- 增加 `shownFlippedIds` 与 `animatingFlipIds`，只让本次新翻开的牌播放翻转动画。
- 已经翻开的牌在后续棋盘重绘时增加 `no-flip-animation`，避免因 DOM 重建重复执行翻牌动画。
- 补充 smoke 用例，覆盖“旧牌不重播翻转，新牌保留翻转动画”。
