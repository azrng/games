## Context

玩家反馈吃子操作没反应。需要检查移动验证逻辑并添加操作反馈。

## Goals / Non-Goals

**Goals:**
- 确保选择己方棋子后能正常吃相邻敌方棋子
- 添加操作失败时的视觉提示

**Non-Goals:**
- 不改变游戏规则

## Decisions

### 1. 检查 isValidMoveTarget 逻辑

当前逻辑：
1. 检查是否相邻
2. 检查是否己方棋子 → 返回 false
3. 检查是否已捕获 → 返回 true
4. 检查是否未翻开 → 返回 false
5. 战斗检查

问题：步骤 2 只检查了己方棋子，但没有明确检查敌方棋子。如果 target.owner 是 undefined 或其他值，可能导致问题。

### 2. 添加操作反馈

当玩家尝试无效移动时，添加视觉反馈（如抖动动画）。

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 修改验证逻辑可能影响其他功能 | 仔细测试各种场景 |
