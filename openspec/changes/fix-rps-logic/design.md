## Context

动物翻翻棋游戏的猜拳先手功能存在逻辑 bug。当前实现中，玩家 A 选择后直接判定胜负，没有等待玩家 B 选择。问题出在事件处理和状态管理上。

## Goals / Non-Goals

**Goals:**
- 修复猜拳逻辑，确保等待两位玩家都选择后再判定胜负
- 优化猜拳 UI 交互，明确区分两位玩家的选择阶段
- 添加选择状态重置功能

**Non-Goals:**
- 不改变游戏的核心规则
- 不添加 AI 对手功能

## Decisions

### 1. 状态管理：使用明确的阶段标记

**选择**: 使用 `rpsPhase` 变量区分 'a-turn' 和 'b-turn' 阶段
**理由**: 比检查 `rpsChoices.a` 是否存在更清晰，避免状态歧义
**替代方案**: 继续使用现有逻辑（存在 bug）

### 2. 事件处理：禁用已选择按钮

**选择**: 玩家 A 选择后禁用所有按钮，等待玩家 B 选择
**理由**: 防止重复点击，明确区分两位玩家的选择阶段
**替代方案**: 仅视觉标记选中状态（可能导致误操作）

### 3. UI 反馈：明确显示当前选择者

**选择**: 在状态文本中明确显示"等待玩家 A 选择"或"等待玩家 B 选择"
**理由**: 让玩家清楚知道当前是谁的回合
**替代方案**: 仅显示"等待选择"（不够明确）

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 玩家可能不理解需要两人分别选择 | 添加清晰的提示文本 |
| 选择后忘记是谁的回合 | 在 UI 中明确显示当前玩家 |
